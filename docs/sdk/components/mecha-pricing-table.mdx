---
title: "ArcaPricingTable"
description: "A high-fidelity pricing table with automated checkout integration."
---

The `ArcaPricingTable` is the flagship UI component of the SDK. It fetches your plan details (name, duration, tiers, features) and displays them in a premium, interactive interface.

## Usage

```tsx
import { ArcaPricingTable } from 'arca-react';

function MyPricing() {
  return (
    <ArcaPricingTable 
      planId="0x77c4..." 
      userId="user_99" 
      recommendedTierId="tier_premium"
    />
  );
}
```

## Props

<ParamField prop="planId" type="string" required>
  The unique ID of the plan you want to display.
</ParamField>

<ParamField prop="userId" type="string">
  The ID of your user. This will be passed to the smart contract as `buyerData`, allowing you to link the on-chain subscription to your off-chain user database.
</ParamField>

<ParamField prop="redirectUrl" type="string" default="window.location.href">
  The URL the user should be redirected back to after a successful payment.
</ParamField>

<ParamField prop="appearance" type="ArcaAppearance">
  Customization object for theming. See [Theming](/sdk/theming) for details.
</ParamField>

<ParamField prop="hideBranding" type="boolean" default="false">
  Set to `true` to remove the "Powered by Arca" footer.
</ParamField>

<ParamField prop="recommendedTierId" type="string">
  The ID of the tier that should be highlighted as "Recommended". Defaults to the second tier in the list.
</ParamField>

<ParamField prop="className" type="string">
  Standard CSS class name applied to the outer container.
</ParamField>

<ParamField prop="style" type="React.CSSProperties">
  Standard inline styles applied to the outer container.
</ParamField>

<ParamField prop="customLabels" type="ArcaLabels">
  Custom label overrides for UI text elements. Supports translation or custom phrasing.
  
  ```typescript
  interface ArcaLabels {
    activeSubscription?: string;      // Default: "Current Plan"
    upgrade?: string;                 // Default: "Upgrade"
    downgrade?: string;               // Default: "Downgrade"
    changePlan?: string;              // Default: "Switch Plan"
    getTier?: string;                 // Default: "Get {{tierLabel}}"
    selectPlanDescription?: string;   // Default: "Select a membership tier to access the protocol."
    subscriptionActiveHeader?: string; // Default: "Subscription Active · {{countdown}} Left"
  }
  ```
</ParamField>

<ParamField prop="classNames" type="ArcaClassNames">
  Custom TailwindCSS or standard class overrides for styling table elements.
  
  ```typescript
  interface ArcaClassNames {
    container?: string;
    grid?: string;
    card?: string;
    badge?: string;
    tierLabel?: string;
    priceContainer?: string;
    priceAmount?: string;
    priceMuted?: string;
    featuresList?: string;
    featureItem?: string;
    featureTitle?: string;
    featureDescription?: string;
    button?: string;
  }
  ```
</ParamField>

<ParamField prop="renderHeader" type="(plan: Plan, subscription: ArcaSubscription) => React.ReactNode">
  A custom render function to completely replace the default pricing table header section.
</ParamField>

<ParamField prop="renderFooter" type="() => React.ReactNode">
  A custom render function to completely replace the default footer / brand section.
</ParamField>

<ParamField prop="renderTierButton" type="(tier: Tier, state: { isActive: boolean; isCurrent: boolean; isDisabled: boolean; isUpgrade: boolean; isDowngrade: boolean; label: string }, handleSelect: () => void) => React.ReactNode">
  A custom render function to completely replace the default CTA button for each tier. Allows injecting your own custom button designs or logic.
</ParamField>

## Visual Features

- **Automated Fetching**: Retrieves all tiers, prices, and features from IPFS.
- **Member Awareness**: If the provided `userId` has an active subscription, the table will highlight their current plan and show a countdown timer.
- **Responsive Design**: Fluidly adapts from mobile to desktop layouts.
- **Animations**: Uses native CSS transitions for smooth hover states and entry animations.
