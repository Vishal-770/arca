---
title: "ArcaProvider"
description: "The core configuration component for the Arca SDK."
---

The `ArcaProvider` is a context provider that should wrap your entire application. It stores your API key and portal settings, enabling hooks like `useArca` and components like `ArcaPricingTable` to function correctly.

## Props

<ParamField prop="apiKey" type="string" required>
  Your Arca Developer API Key. Used for authenticated requests to the protocol indexers.
</ParamField>

## Implementation Example

```tsx
import { ArcaProvider } from 'arca-react';

export default function App() {
  return (
    <ArcaProvider 
      apiKey={process.env.NEXT_PUBLIC_ARCA_API_KEY}
    >
      {/* Rest of your application */}
    </ArcaProvider>
  );
}
```

<Warning>
  While the `apiKey` is required, ensure you are using it in a way that aligns with your security needs. For client-side SDK usage, we recommend scoped keys that only have read access to plan data.
</Warning>
